# 🧙‍♂️ ZFS Storage Repair Wizard (Proxmox 9)

Интерактивный bash-скрипт с удобным текстовым интерфейсом (TUI) для автоматизации, безопасной замены и ребилда дисков в массивах ZFS на серверах Proxmox VE 9.x (OpenZFS 2.2+).

Скрипт полностью исключает человеческий фактор: защищает от случайного затирания рабочих данных, автоматически обходит системные блокировки ядра Linux и собирает массивы по эталонным стандартам разметки Proxmox (по ID разделов с сохранением скрытого буфера `part9`).

---

## 🔥 Ключевые возможности

- **Универсальность**: Автоматически распознает тип архитектуры пула — **Mirror (RAID-1)**, **RAIDZ1 (RAID-5)**, **RAIDZ2 (RAID-6)** и **RAIDZ3**.
- **Два режима работы**: 
  - `REPLACE` — стандартная замена диска внутри существующего зеркала или RAIDZ.
  - `ATTACH` — умное расширение одиночного диска (Stripe) до полноценного зеркала (Mirror).
- **Информативное меню**: При выборе диска выводит не только имя устройства и объем, но и **тип шины (SATA/NVMe/USB)**, а также заводской **серийный номер (SN)** для точной физической идентификации накопителя в сервере.
- **Графический Прогресс-бар**: Встроенный интерактивный индикатор выполнения ребилда (Resilver) в реальном времени показывает процент выполнения, скорость сканирования (MB/s) и точное расчетное время до окончания.
- **Умный обход блокировок**: Автоматически подавляет ошибки ядра `Device or resource busy` и `invalid vdev specification` путем принудительной очистки старых метаданных (`wipefs`, `labelclear`) и сброса vdev-кэша.
- **Подробное логирование**: Каждый шаг работы, включая детальный вывод консольных команд, записывается в изолированный файл `/var/log/zfs-repair.log`, который автоматически очищается при каждом новом запуске.

---

## 🛠 Установка зависимостей на сервере

Перед использованием мастера убедитесь, что на вашем сервере Proxmox установлены необходимые утилиты отрисовки интерфейса и чтения разделов:

```bash
apt update && apt install dialog parted -y
```

## 📸 Как это выглядит в работе

1. **Главное меню**: Скрипт сканирует систему и выводит список только тех пулов, которые требуют внимания (`DEGRADED` или одиночные Stripe).
2. **Анализ накопителей**: Мастер сам находит виновника аварии (vdev ID или имя диска) и предлагает подтвердить замену.
3. **Безопасный выбор**: Вы видите список свободных дисков с их серийными номерами — перепутать накопители невозможно.
4. **Ребилд**: Запускается автоматическое клонирование геометрии разделов эталонного диска и активируется скрытый тюнинг скорости ядра (`zfs_resilver_min_time_ms`), ускоряющий восстановление в десятки раз.
